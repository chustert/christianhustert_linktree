Lean Framework Style Guide

Keeping HTML structure lean and maintainable

The aim is to keep Webflow's generated HTML structure visible to and rendered by browsers as lean as possible whilst keeping a Webflow page easy to maintain and organise and at the same time provide an accessible website.The layout strategy, which uses a 12 grid layout and primarily flexbox to ensure responsiveness, is based on the Bootstrap CSS framework but has been adapted to Webflow's "desktop first" approach.

Layout

  1. Header, Main and Section are using Webflow's custom element with the appropriate tags.
  2. Header and Main usually don't need custom class names as they serve as wrappers for a clear HTML semantic
  3. If a Section needs custom styling, keep the naming consistent with .section-title, e.g. .section-main
  4. A section's direct children are containers. Read more about containers below.
  5. The Footer always sits outside the Main element! To style the footer element, apply a class to the footer tag, e.g. .footer. A footer can have multiple containers.

The core structure strategy aligns with best practices to keep a flat hierarchy and clean DOM whilst keeping a Webflow page easy to maintain and organise (ie. using a page-wrapper and a "Global Styles" embed).

Webflow's Breakpoints

Webflow has 4 preset breakpoints:

  1. Desktop (992 and up)
  2. Tablet (991px and down)
  3. Mobile Landscape (767px and down)
  4. Mobile Portrait (478px and down)

You also have the option to add 3 larger breakpoints:

  1. 1280px (1280px and up)
  2. 1440px (1440px and up)
  3. 1920px (1920px and up)

Containers

Container breaking

Containers are direct children of Sections.
The default Container class has its own styles. To change styles, duplicate the container class and give it a new name with the naming convention Container Title, e.g. "Container Footer" or "Container Contact". If you have a combo class applied to your container class (e.g. "Tablet"), make sure you apply your new container styles to the new container class only and not to both classes. See "Container Footer" at the bottom of this page as an example.
While containers can be nested, most layouts do not require a nested container.

Each container can be assigned a class that specifies the breakpoint when the container scales to 100% width. Default is 100%.
For example, Container and Tablet indicate that the container remains 991px or 100% wide (whatever is smaller), meaning on Desktop or anything above remains at a width of 991px and has a width of 100% on all devices smaller and including Tablet.

Grid

  • Use Grid when you're working with a two-dimensional layout, such as aligning items both horizontally and vertically.
  • It's great for overall page layout where you have distinct rows and columns.
  • Grid is especially useful for complex layouts where you need precise control over the placement of items within a grid structure.
  • It's well-suited for building large-scale layouts, such as entire webpage structures.

Lean As Framework does not have grid structures pre-defined, instead we're using a naming convention.
We start by adding the grid element to the Designer and giving it the class Grid Row; this loads the predefined styles. Add a combo class resembling the grid structure; e.g., for a 1fr 5fr grid, we add the class name 1fr 5fr (see example below).
Because grids are mainly used for overall page layout, we would just be adding normal divs with custom class names to the grid element. Make sure to give the divs the correct tags, such as aside or main (see example below).

Flex

  • Flexbox is best used for arranging items along a single axis, either horizontally or vertically.
  • Use Flexbox for smaller-scale layout tasks like aligning items within a container or creating responsive navigation menus.
  • It's excellent for distributing space within a container and aligning items in a row or column.
  • Flexbox is particularly useful for creating flexible and responsive designs that adapt well to different screen sizes and device orientations.

Column gaps

To align content, we set padding-right and padding-left on each Column and offset them with a negative margin at the beginning and end of each Row. To change the column gap, go to Webflow's Variables (V), navigate to the "flex column gaps" folder, and change the size value of flex "gap" and "n_gap" to the same size value but negate the value you enter for "n_gap" (for example, gap = 0.5rem and n_gap = -0.5rem). The actual gap between columns is the entered gap size multiplied by two (for example, enter 0.5rem and the actual gap size is 1rem).

Column breaking

Each column can be assigned a class that specifies the breakpoint when the column scales to 100% width. Class naming is * [Breakpoint] Columns.
For example, "7 Tablet Columns" indicates that the column remains 7 columns wide on tablets and any smaller screen sizes, unless another class like "12 Mobile Portrait Columns" is applied.

Offsetting columns

You can offset columns with the Offset [Breakpoint] * classes. For example, "Offset 3" offsets the element by three columns to the right, whereas "Offset Tablet 3" offsets the element by three columns to the right on Tablet and below; no offset is applied to greater breakpoints.
You can also combine multiple offset classes to make the column behave different for each breakpoint, e.g. "Offset Tablet 4" and "Offset Mobile Landscape 2".

What’s a Rich Text element?

The rich text element allows you to create and format headings, paragraphs, blockquotes, images, and video all in one place instead of having to add and format them individually. Just double-click and easily create content.

  • list item

Static and dynamic content editing

A rich text element can be used with static or dynamic content. For static content, just drop it into any page and begin editing. For dynamic content, add a rich text field to any collection and then connect a rich text element to that field in the settings panel. Voila!

How to customize formatting for each rich text

Headings, paragraphs, blockquotes, figures, images, and figure captions can all be styled after a class is added to the rich text element using the "When inside of" nested selector system.
